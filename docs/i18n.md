# Internationalization (i18n)

TypeRP Framework is multilingual by design.

## Model

- The server selects an **active locale** at startup.
- Per-player runtime language switching is **not required** in v1.
- Core, modules, plugins, and compatibility layers must provide their own translations.
- Namespaces are used per module to prevent collisions.
- There is a fallback locale in case keys are missing.

## Packages

The `packages/i18n` package is the shared foundation to retrieve and process keys for the active locale. Each FiveM resource / NUI app uses this package to resolve the server-selected language strings.
